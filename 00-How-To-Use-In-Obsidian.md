# How To Use This Vault In Obsidian

## Open
1. Install Obsidian.
2. `Open folder as vault` → pick `mern-interview-vault`.
3. Start at [[README]] (the MOC = Map of Content).

## Links
- `[[Note-Name]]` = internal link. Click to jump.
- Graph view (left sidebar icon) shows all connections.
- Backlinks panel (right) shows what points to current note.

## Note format (every note follows this)
- **Q** — the interview question.
- **A** — short senior answer (say this out loud in interview).
- **Code** — example. DSA gives *with inbuilt* + *without inbuilt* versions.
- **How** — mechanism.
- **Why** — why it works / why it matters.
- **Where / Scenario** — real production use case.

## Study method
1. Read note.
2. Cover code. Rewrite from memory.
3. Say the **A** out loud like real interview.
4. Tag weak notes: add `#revise` anywhere in file → search `#revise` later.

## Tags to add yourself
- `#revise` — come back.
- `#done` — solid.
- `#asked` — actually got asked in a real interview.

## Suggested plugins
- Advanced Tables — edit md tables easy.
- Dataview — query notes.
- Excalidraw — draw system design diagrams.
